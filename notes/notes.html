HTML Structure Part 2

  • The HTML document can be broken down into 3 parts:
    1. DOCTYPE: Describes type of HTML, while there are technically different types, for majority of the HTML you'll write, <!DOCTYPE html> will be fine
    2. <head>: Describes meta information about the site, such as the title, and provides links to scripts and stylesheets the site needs to render and behave correctly

  • It is good practice to include all the basic parts of the template so that you are not relying on the guesswork of browsers to display your sites correctly

Starting Small

Defining the Viewport

  • The viewport defines the area of the screen that the browser can render content to

Pixels, pixels and moar pixels!

  • A device-independent pixel, also known as density-independent pixel, dip, dp, is a physical unit of measurement based on a coordinate system held by a computer and represents an abstraction of a pixel for use by an application that an underlying system then converts to physical pixels

Quiz: Pixelation

  • Tech specs will refer to the number of hardware pixels

Quiz: Calculating DPR

  • To calculate the device pixel ratio, using one-dimensional space divide the hardware pixels
  • With one dip for every two hardware pixels, the device pixel ratio is 2

Quiz: What's the difference?

  • The viewport and the device pixel ratio are both likely causes for the differences between devices

Quiz: Calculating CSS Pixels

  • Divide the hardware px by the DPR to calculate the CSS px

Quiz: How wide is the viewport?

  • To calculate the viewport divide the px width by the DPR

Setting the Viewport

  • Use a meta tag in the head to set a one to one relationship between device independent pixels and CSS pixels

Large Fixed Width Elements

  • Use relative positions, instead of absolute positions, such as width 100%

Max-width on elements

  • Use max-width to have elements like images, videos, and objects be more responsive

Quiz: Relative Sizes

  • Percentages are relative and are responsive
  • Smaller elements may be fine depending on the size

Tap Target Sizes

  • Fingers and thumbs are roughly 10mm so the minimum tap target size should be 40px by 40px or ideally 48px by 48px for those with fatter thumbs

Quiz: Tap Targets

> Building Up

Lesson Intro

  • When designing responsively, it is important to consider the layout of the page depending on the size of the screen

Basic Media Query Intro

  • Media queries, provide easy logic for applying different styles depending on device characteristics, like its width, height, or device pixel ratio

Adding a Basic Media Query

  • To add a media query link a new stylesheet and set the media attribute
    • <link rel="stylesheet" media="screen and (min-width:500px)" href="over500.css">
    • Stick with either screen or print for the media attribute value

Adding a Basic Media Query 2

  • There are two other ways you can apply media queries:
    • Embed them with an @media tag, or
    • Embed them with an @import tag
  • For performance reasons, you definitely want to avoid @import
  • Linked CSS have many small http requests while @media have few big http requests

Next Step Media Queries

  • Use either min-width or max-width for media queries
  • Avoid using min-device-width and max-device-width because it uses the device's width instead of the browser's width

> Common Responsive Patterns

Intro to Patterns

  • Responsive patterns are constantly evolving but there are some established patterns that are used
  • Four commonly used patterns are:
    1. Mostly Fluid
    2. Column Drop
    3. Layout Shifter
    4. Off Canvas

Pattern - Column Drop

  • Column drop is probably the easiest
  • At its narrowest viewport each element simply stacks vertically
  • As things get wider, the elements expand, at the first breakpoint two elements are now side by side with the third element is underneath
  • At the second breakpoint, three elements are now side by side and as the viewport gets wider margins are now added instead

Pattern - Mostly Fluid

  • The mostly fluid pattern is very similar to column drop, but it tends to be a bit more grid like
  • It has more columns and with columns fitting in different ways, depending on the viewport width

Quiz: Mostly Fluid Part 1

  • Flexbox is such a powerful tool for responsive layout, keep playing with it!

Quiz: Mostly Fluid Part 2

  • Mastering flexbox and grid is crucial to responsive web design

Quiz: Combining Fluid Layouts

  • See pattern-mostly-fluid-blankcss.html file in the supporting materials folder

Pattern - Layout Shifter

  • The layout shifter pattern is probably the most responsive pattern with multiple break points across several different screen widths, but the key to this layout is the way that content moves about instead of reflowing and dropping below other columns
  • Remember, the default order value for an element is zero

Quiz: Which is Which?

  • order and width are super useful, keep using them!

Pattern - Off Canvas

  • With off canvas, instead of stacking content vertically, the off canvas places less frequently used content, for example navigation or app menus, off screen, only showing if the screen is large enough

Quiz: Off Canvas Visualization

  • Understanding the way your elements move is essential
